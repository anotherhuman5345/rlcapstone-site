Catching AI-driven database ransomware

This is the flip side of my AI hacking agent — a defense tool. It reads a database log and flags the signs of JADEPUFFER, the first known case of ransomware run entirely by an AI. Paste a log, hit scan, and it points out what looks dangerous.

Defensive and for learning. This tool only reads and scores a log — it never runs anything and never touches a real database. It's built to help spot an attack, not to do one.

What is JADEPUFFER?

Security researchers at Sysdig found the first ransomware attack driven start-to-finish by an AI (a large language model). It breaks into a company's database, quietly makes a backdoor admin account, scrambles all the settings with encryption, deletes the originals, and leaves a table demanding Bitcoin. The scary-cool part: the AI adapts on the fly — when a step fails, it diagnoses the error and tries a different way in seconds, and it even leaves little reasoning notes in its code, like "high-ROI databases to drop." My detector looks for exactly those tells.

What it looks for

The tool grades a log out of 100 using the behaviors from the report, each tied to its place in the attack:

  • The ransom table — a table named like a ransom note, or one holding a Bitcoin address, a contact email, or "your data has been encrypted."
  • Encrypt-then-destroy — copying a table through AES encryption and then dropping the original, plus turning off foreign-key checks to force mass deletes.
  • A backdoor admin — a new admin user shoved straight into the login tables.
  • Breakout probing — using the database to read host files like /var/run/docker.sock or list special functions.
  • The "agentic" tell — natural-language reasoning left in the SQL, which a human hacker would never write but an AI does. This is the newest and most interesting signal.
  • Known indicators — the exact addresses, emails, and table names from the report.

Why this is the v2

The first version of this project was an AI that attacks a practice site. As AI-run attacks like JADEPUFFER start showing up for real, the more useful skill is catching them — so v2 turns the idea around into a detector. Next steps would be reading real database logs live and scoring a whole session as it happens.
