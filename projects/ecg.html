Why "accuracy" is a trap here

About 90% of the beats in this data are normal. So a lazy model that calls every beat "normal" already scores 90% — while catching exactly zero of the dangerous beats that actually matter. That's the worst possible model wearing a good-looking score. So instead I track how well it does across all four types fairly, which only goes up when it handles the rare, important beats too.

Turning the signal into beats

Each recording is a half-hour ECG with a doctor's mark on every single heartbeat. I chop it into one short window per beat and sort each into four types. The most important decision was how to split the data: I split it by patient, not by beat. If you split by beat, the same person's heartbeats end up in both the training and testing piles, and the model just learns to recognize that person instead of learning about heart problems — which is exactly the trap the next part is about.

Watching it learn (and the catch)

Here's the story in one chart. While training, its fair-score climbed past 96% and looked great. But that practice test shared patients with the training data. On the real test — 22 patients it had never seen — the honest fair-score drops to 44%. The gap between that climbing line and the real number is the whole lesson.

What it learned, and what it didn't

The breakdown is where it gets interesting. On new patients it catches 86% of one dangerous type (ventricular) — actually useful — but only 9% of another type (supraventricular). That's not random; it's a clue.

A ventricular bad beat looks weirdly shaped, so a model that reads shapes spots it easily. But a supraventricular bad beat looks almost exactly like a normal one — the thing that makes it abnormal is its timing (it comes early). My model looks at each beat all by itself with no sense of timing, so it's basically blind to the one clue it needs for the type it fails on.

What's next (v2)

The fix comes straight from the problem: give the model timing. If I feed it how far apart the beats are, it could finally notice when a beat comes early — the giveaway for that missed type. That's the next version, and its result will show up in the table above so you can see if the idea works.
